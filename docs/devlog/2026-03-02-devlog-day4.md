## Summary

Day 4 was a lighter testing day focused on battery life benchmarking and Bluetooth audio. No SIM or SMS testing — that's pushed to tomorrow when a top-up is available. The headline finding is that battery life is better than expected under active use, with Waydroid having less impact than anticipated. pmbootstrap was also set up successfully and produced a first clean build in under 10 minutes.

---

## User Experience Observations

- **AirPods pair easily and stay connected.** No manual pairing steps beyond the standard Bluetooth menu. Connection was stable throughout 30 minutes of active use. Good result for consumer usability.
- **AirPod settings not accessible.** Transparency mode, ANC, and other Apple-specific controls are unavailable — expected, as these require Apple's proprietary protocol which Linux doesn't support. Worth noting in docs as a known limitation for Apple accessory users.
- **Volume increments need tuning.** Volume steps feel too coarse — workable but not polished. Worth investigating whether this is a PulseAudio or GNOME Mobile issue.
- **Status bar overflow.** When Bluetooth, WiFi, and other indicators are active simultaneously, the battery percentage gets pushed off the right edge of the screen. A real consumer UX issue — users need to see their battery level at all times.
- **Battery reporting anomaly.** A sudden drop from ~55% to 5% was observed during the day. The phone charged normally from 5%, and subsequent tests showed stable reporting. Likely a one-off fuel gauge recalibration event rather than genuine drain. Not repeated.

## Battery Testing

Three test scenarios conducted. All at 50% screen brightness with AirPods connected via Bluetooth.

### Overnight Standby (from Day 3)
- **Conditions:** Screen off, WiFi + mobile data on, no active apps
- **Drain:** 13% over 7h 42m
- **Rate:** ~1.7% per hour
- **Extrapolated standby:** ~20 hours from full charge

### Medium Active Use
- **Conditions:** YouTube playing, AirPods audio, screen on
- **Duration:** 30 minutes exactly
- **Drain:** 10% (63% → 53%)
- **Rate:** ~20% per hour
- **Extrapolated active life:** ~5 hours from full charge

### Heavy Active Use
- **Conditions:** YouTube playing, AirPods audio, screen on, Waydroid open with WhatsApp in background
- **Duration:** 30 minutes exactly
- **Drain:** 10% (83% → 73%)
- **Rate:** ~20% per hour
- **Extrapolated active life:** ~5 hours from full charge

### Key Finding
Medium and heavy tests produced identical drain rates — Waydroid running in the background had negligible impact on battery consumption compared to screen-on YouTube playback. Post-test idle with Waydroid open drained at ~5% per hour, which is acceptable.

### Assessment
5 hours active use from full charge is below target (8 hours) but genuinely usable for moderate daily use. The most likely cause of higher than expected drain is incomplete suspend/deep sleep implementation on Qualcomm under postmarketOS — a known platform-wide issue, not specific to MahalaOS.

## pmbootstrap First Build

Successfully set up pmbootstrap on Fedora and produced a first clean fajita image.

**Setup:**
```bash
git clone https://gitlab.postmarketos.org/postmarketOS/pmbootstrap.git
cd pmbootstrap
./pmbootstrap.py init
```

**Config:**
- Channel: edge
- Device: oneplus-fajita (fajita)
- UI: gnome-mobile
- Audio backend: pulseaudio
- WiFi: wpa_supplicant
- Systemd: yes
- Hostname: oneplus-fajita
- Locale: en_GB.UTF-8
- SSH keys: copied

**Build time:** ~7 minutes (pre-built binary packages, no compilation required)

**Image location:**
`/home/sidalous/.local/var/pmbootstrap/chroot_native/home/pmos/rootfs/oneplus-fajita.img`

**Notes:**
- Image built without FDE (full disk encryption) — `postmarketos-base-nofde`
- pip install of pmbootstrap no longer works — must clone from GitLab directly
- Extra packages field must be left blank (Enter) — typing `n` registers as a package name and breaks the build

**Next steps for overlay:**
The image is clean vanilla postmarketOS. MahalaOS fixes need to be layered in before flashing:
1. `mahala-waydroid-net.service` — Waydroid firewall fix
2. `mahala-call-audio.service` + script — call audio profile switcher
3. `pipewire-tools` and `pipewire-echo-cancel` packages
4. `/etc/unl0kr.conf.d/` cleanup (remove incorrect config from earlier attempts)
